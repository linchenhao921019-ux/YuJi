# 余迹

原生 SwiftUI macOS 卸载残留与应用缓存清理工具，视觉适配 macOS 26。

当前公开源码版本：`v2026.07.14`。公开仓库使用示例 Bundle ID `com.example.yuji`；本地构建时可替换为自己的反向域名标识。

## 功能

- 扫描 `~/Library` 与 `/Library` 中的应用支持、缓存、WebKit、HTTP 存储、日志、状态、偏好设置和 Cookies 线索
- 分批测量候选目录，显著减少 `du` 进程启动次数，并在固定时间预算内覆盖更多位置
- 同一次扫描中识别已安装应用在 `~/Library/Caches` 下超过 1 MB 的缓存，并通过现有选择与废纸篓流程清理
- 为避免 macOS 26 每次弹出“访问其他 App 数据”，常规扫描不会进入 `Containers` 与 `Group Containers` 私有容器
- 将目录名称与当前已安装应用名称及 Bundle ID 交叉匹配
- 根据目录类型、更新时间、体积与应用标识特征给出风险与置信度
- 支持搜索、风险筛选、大型残留、白名单和扫描历史
- 外观自动跟随 macOS，在系统深色模式下同步切换
- 使用原生 macOS 毛玻璃窗口背景，并保持内容区域清晰可读
- 检测超过 30 天、仅含系统占位文件的空白残留文件夹，并单独标记为“需确认”
- 使用 macOS 系统废纸篓 API 安全清理，可从废纸篓恢复
- 扫描与清理阶段双重保护 Apple 系统目录、备份、通话记录和隐私数据库
- 自动识别书签、登录信息、会话、存档、钱包等个人数据特征，并在详情与清理确认中醒目提醒
- “全选”选择当前列表中的所有项目；风险等级仅作提醒，不替用户限制选择
- 清理时再次验证路径范围、系统保护规则和符号链接，避免扫描结果被替换后误伤其他内容
- 导出 CSV 扫描报告
- 快捷打开“完全磁盘访问”设置

## 构建

```bash
chmod +x scripts/build-app.sh
./scripts/build-app.sh
```

构建产物位于 `dist/余迹.app`。构建脚本会优先使用本机稳定签名身份；未配置时才回退到临时 ad-hoc 签名。对外分发需要 Apple Developer ID 签名与公证。

首次进行本机长期使用前，可运行 `scripts/create-local-signing-identity.sh` 创建仅供本机使用的稳定签名身份。之后构建脚本会自动使用该身份，使完全磁盘访问授权在覆盖更新后仍能匹配同一个应用。对外分发仍需要 Apple Developer ID 签名与公证。

若项目存放在 Google Drive，请运行根目录的 `install_app_to_applications.sh`。脚本会复制应用到 `/Applications`，清理 Drive 附加的扩展属性并重新进行本地签名。

## 安全说明

扫描算法采取保守策略，系统组件和已安装应用目录会被排除；近期更新、名称泛化或可能包含用户数据的候选会标记为“需确认”。任何自动判断都可能出现误判，清理前应检查路径与证据。
